Local GitHub API broker

Least-privilege GitHub access for LLMs.

Marlow is a small local HTTP service that lets an LLM read and write a fixed set of private repositories — and nothing else. Every request is validated, every repository is checked against a closed allow-list, and the token is held only in memory.

The security model

The whole point is what it won't do.

Marlow exists to hand an autonomous agent a key that only opens two doors. The boundary is enforced in the domain layer, before any network call or credential read — not bolted on at the edge.

01

Closed allow-list

Only nrwl/ocean and nrwl/nx are reachable. Any other repository is rejected with 403 repo_not_allowed before GitHub is ever contacted.

02

The token only lives in memory

Marlow reads the token once from MARLOW_GITHUB_TOKEN and holds it in memory — never logged, never written to disk.

03

Writes require confirmation

Creating an issue, comment, or pull request — or closing one — requires { "confirm": true } in the body. No accidental mutations.

04

No raw GitHub payloads

Every response is mapped to a small internal DTO. Upstream fields and error text never pass straight through to the caller.

05

Bring your own secret source

Marlow asks only for a token. Fetch it however you trust — pass-cli, 1Password, Vault, a CI secret — Marlow neither knows nor cares where it came from.

06

Sanitized failures

Credential and upstream errors surface a stable code and status — never a stack trace, a secret, or the raw request.

How it works

Dependencies point inward.

A DDD-lite layout: pure rules at the centre, infrastructure at the edge, and an app that wires them together. The domain knows nothing about HTTP, GitHub, or where the token comes from.

Layers · outer depends on inner
  • marlow-api Fastify · composition root
  • api-contracts · api-errors Zod schemas · error vocabulary
  • infrastructure-github Octokit adapter · DTO mapping
  • infrastructure-credentials static token provider
  • application use cases · ports
  • domain allow-list · value objects · Result

Module boundaries are enforced by Nx tags — infrastructure may depend on the application and domain, never the reverse.

Request lifecycle

  1. Validate input
    Zod parses params, query, and body.
  2. Build the value objects
    RepoRef, GitRef, GitSha, FilePath, …
  3. Check the allow-list
    A disallowed repo stops here — 403, before any token or network call.
  4. Resolve the token
    read from the environment, held in memory.
  5. Call GitHub
    Through the Octokit adapter.
  6. Map to a DTO & respond
    JSON only — no raw payloads.

Run it

One command in development.

Marlow is local-first. In development it rebuilds and serves; in production it bundles to a single self-contained file.

Develop

# install workspace dependencies
pnpm install

# rebuild + serve the API (dev)
pnpm nx serve marlow-api

Build & serve

# self-contained production bundle
pnpm nx build marlow-api

# run it (with the env below set)
node apps/marlow-api/dist/main.js

Verify

pnpm nx run-many -t typecheck test lint

Marlow needs the token at startup and reads it from MARLOW_GITHUB_TOKEN. How it gets there is entirely up to you.

Inject it for the process only with pass-cli run -- node …, substitute it from any secret manager, or export it directly — Marlow never spawns a secrets tool itself.

Bind address defaults to 127.0.0.1. Keep it on loopback unless you put your own authentication in front.

Configure

Everything through the environment.

Configuration is read once at startup. The only credential Marlow needs is MARLOW_GITHUB_TOKEN — provide it however you like. Everything else has a sensible default.

Variable Required Default Description
MARLOW_GITHUB_TOKEN required — GitHub token. Supply it however you like — see below.
MARLOW_HOST optional 127.0.0.1 Bind address. Keep on loopback by default.
MARLOW_PORT optional 3000 Port to listen on.
NODE_ENV optional — production suppresses 5xx detail in responses.
MARLOW_GITHUB_USER_AGENT optional marlow User agent sent by the Octokit client.
MARLOW_GITHUB_BASE_URL optional — GitHub API base URL, e.g. for GitHub Enterprise.

Mint the token with its permissions pre-filled. A fine-grained token is least-privilege — it pre-selects Contents (read), Issues and Pull requests (read/write), and Commit statuses (read) for the nrwl owner; on the page, also add Checks (read) and limit access to nrwl/ocean and nrwl/nx. A classic token is simpler but coarser — the single repo scope reaches the private repository on the allow-list.

Marlow only reads MARLOW_GITHUB_TOKEN — supply it however you trust: 

# inject it for the process lifetime via Proton Pass
pass-cli run -- node apps/marlow-api/dist/main.js

# or substitute from any secret source
MARLOW_GITHUB_TOKEN="$(pass-cli item view <ref> --field token)" node apps/marlow-api/dist/main.js

API reference

One namespace, scoped to the allow-list.

Every repository route lives under /repos/:owner/:repo and only answers for an allow-listed repository. Errors share one shape: { "error": { "code", "message", "details?" } }.

Method Endpoint Does
GET /openapi.json OpenAPI 3.1 description of this API — runtime discovery.
GET /health Liveness check.
GET /repos List the allow-listed repositories.
GET /repos/:owner/:repo/permissions Token's permission level on the repo.
GET /repos/:owner/:repo/tree List the tree at a ref (?ref=&recursive=).
GET /repos/:owner/:repo/contents/* Read a file (?ref=).
GET /repos/:owner/:repo/search/code Search code, scoped to the repo (?query=).
GET /repos/:owner/:repo/commits List commits (?ref=&path=).
GET /repos/:owner/:repo/commits/:sha A commit with stats and files.
GET /repos/:owner/:repo/issues List issues (?state=).
GET /repos/:owner/:repo/issues/:n Get one issue.
POST /repos/:owner/:repo/issues Create an issue. confirm
POST /repos/:owner/:repo/issues/:n/close Close an issue. confirm
GET /repos/:owner/:repo/issues/:n/comments List issue comments.
POST /repos/:owner/:repo/issues/:n/comments Comment on an issue. confirm
GET /repos/:owner/:repo/pulls List pull requests (?state=).
GET /repos/:owner/:repo/pulls/:n Get one pull request.
